# 408 考研知识点卡片小程序 - V1 (MVP) 开发计划

## 产品愿景 (Vision)

打造一个极简、学术风的 408 考点背诵工具。通过**卡片化交互**解决厚重复习资料难以携带、知识点难以拆解的痛点。V1 阶段完全开放内容，通过极致的渲染效果（LaTeX/代码）树立专业口碑



## 核心功能模块计划 (Functional Modules)

### 知识大纲导航 (Knowledge Discovery)

**多层级结构**：

- **专业选择**：首屏展示（如：408 计算机综合、308 护理综合）。
- **科目列表**：点击 408 后，展示数据结构、计算机组成原理、操作系统、计算机网络。
- **章节索引**：展示各科目的官方大纲章节（如：OS 的进程管理、内存管理）。

**进度概览**：在科目/章节图标旁显示“本地学习进度”（如：已刷 25/120），数据取自小程序本地缓存。



### 沉浸式卡片引擎 (Immersive Card Engine)

**交互逻辑**：

- **3D 翻转**：点击卡片通过 CSS 3D 动画翻转，正面显示“考点/问题”，反面显示“详细解析”。
- **滑动手势**：左右滑动切换上一题/下一题。

**专业渲染**：

- **公式支持**：完美渲染 LaTeX 数学公式（如时间复杂度 O(nlogn)、地址计算公式）。
- **代码高亮**：对 408 常见伪代码或 C 语言代码进行高亮处理。

**自评反馈**：

- 提供三个按钮：**“不熟”**、**“模糊”**、**“掌握”**。点击后自动保存状态并切题。



### 游客记忆中心 (Local Data Center)

**本地存储策略**：

- 在不登录的情况下，所有点击反馈结果存储在微信小程序的 `LocalStorage` 中。

**错题/重难点本**：

- 系统自动汇总标记为“不熟”和“模糊”的卡片，形成一个“待强化”列表。

**搜索与定位**：

- 全局搜索框，支持通过关键词（如“信号量”、“TCP 拥塞”）快速找到对应卡片。



## 技术实施规范

### 前端 (Uni-app / Vue 3)

**组件化**：

- `CardComponent.vue`：独立封装卡片翻转逻辑，解耦业务与动画。
- `MarkdownParser.vue`：集成 `mp-html` 或 `towxml` 插件，处理复杂的 408 文本。

**性能优化**：

- **数据预拉取**：在进入科目页时，异步拉取该科目下前 50 张卡片到本地内存，确保划动不卡顿。



### 后端

**接口方案**：

- `GET /api/v1/content/majors`：返回全专业目录。
- `GET /api/v1/content/cards?subjectId=xxx`：支持分页返回题目。

**无感鉴权**：

- 后端配置过滤器（Filter），对 `/api/v1/content/**` 下的 GET 请求不进行 JWT 拦截。

**高可用预留**：

- 针对热门章节（如：OS 进程同步、计网协议层），在 Redis 中缓存整个章节的 JSON 数据，减少数据库 IO。
