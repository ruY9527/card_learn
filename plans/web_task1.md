# Web 后台管理系统设计计划 (Admin Portal)

## 系统核心愿景

后台系统旨在解决“高质量 408 考点内容从哪里来”和“内容如何安全管理”的问题。它通过 **AI 赋能**实现卡片的自动化批量生产，并利用 **RBAC 模型**确保管理操作的审计性与安全性



## 功能模块计划 (Functional Modules)

### 身份与安全中枢 (RBAC & Security)

**管理员管理**：管理后台账号，支持重置密码、禁用/启用。

**角色定义**：配置“超级管理员”、“内容编辑”、“学科专家”等角色。

**菜单与权限分配**：细粒度控制。例如：“内容编辑”只能录入 408 卡片，不能查看系统日志。

**操作审计**：记录所有修改业务数据的行为（如：谁在何时修改了“死锁”定义的解析），利用 AOP 实现。



### 知识图谱工厂 (Content Management)

**多专业层级管理**：维护 `Major` -> `Subject` -> `Chapter` 的树状结构。

**卡片库 (Flashcard Bank)**：

- **富文本/Markdown 编辑器**：集成支持 **LaTeX** 实时渲染的编辑器，确保 408 里的数学公式、伪代码在录入阶段就可见即所得。
- **标签管理**：对知识点进行多维度打标（如：#高频考点、#历年真题、#难点）。

**资源管理**：管理题目中涉及的图示（如：存储器层次结构图、TCP 时序图），支持上传到本地或对象存储



### AI 智能工作站 (AI-Powered Module - 核心亮点)

**AI 批量转化器**：

- **功能**：你可以直接将 408 教材的文本或笔记粘贴进来，系统通过 **Qwen API** 自动解析并拆分为“正面提问、反面答案、知识点标签”的 JSON 结构。
- **预览与入库**：AI 生成后，在 Web 端提供对比预览，管理员点击“确认”后批量存入 MySQL。

**解析增强**：针对已有的官方解析，利用 AI 生成“白话版解析”或“速记口诀”，提升小程序端的用户体验。

**支持手动录入**： 针对ai转化，同时也需要提供非ai的手动录入功能



### 数据运营看板 (Analytics & Dashboard)

**学习数据监控**：统计各科目的卡片被刷次数、平均掌握率。

**异常反馈处理**：处理小程序端用户提交的“内容报错”或“解析建议”。

**题库质量分析**：识别出哪些卡片被用户标记为“太难”，驱动内容团队进行优化。



## 后端技术实现逻辑



### 动态权限校验

利用 **Spring Security + JWT**。

1. **加载权限**：在 `UserDetailsServiceImpl` 中加载用户的菜单权限标识。

2. **注解拦截**：在 Controller 层使用 `@PreAuthorize("@ss.hasPermi('biz:card:add')")` 实现按钮级权限控制

   

## 界面设计风格建议

**框架**：Vue 3 + Element Plus。

**布局**：经典的左侧菜单+顶部面包屑。

**体验**：增加“全局搜索”功能，支持管理员通过关键词快速定位到任何一张 408 卡片



## 第一版



1. **基础框架**：完成 RBAC 五张表的 CRUD 及登录逻辑。
2. **内容通道**：完成专业、科目、卡片的管理页面。
3. **AI 助手接口**：集成 Qwen 接口，实现一个简单的文本转卡片功能原型。

该后台的设计充分考虑了你作为开发者对“效率”的追求。既然小程序端是游客模式，这个 Web 后台就是整套系统唯一的“安全堡垒”。
