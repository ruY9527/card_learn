# 408知识点学习卡片系统 - 项目功能总结

## 项目概述

**Card Learn (408知识点学习卡片系统)** - 一个基于闪卡式学习的考研408专业课复习系统，包含三个主要组件：
- **后端 (card-learn-boot)** - Spring Boot API服务
- **小程序 (card-mini)** - 微信小程序移动端
- **管理后台 (card-ui)** - Vue3后台管理系统

---

## 技术栈

| 组件 | 技术 |
|------|------|
| 后端 | Spring Boot 2.7.18 + MyBatis Plus 3.5.3.1 + MySQL 8.0 + Redis |
| 管理后台 | Vue 3 + TypeScript + Element Plus 2.6.3 + ECharts 6.0 + Vite 5.1.6 |
| 小程序 | Uni-app (Vue 3) + UnoCSS + SCSS + mp-html (LaTeX渲染) |
| 安全 | Spring Security + JWT (jjwt 0.12.3) + BCrypt |
| 工具库 | Hutool 5.8.22 + FastJSON 2.0.25 |

### 1.2 学习状态定义

| 状态 | 值 | 含义 | 下次复习间隔 |
|------|-----|------|-------------|
| 未学 | 0 | 未学习 | - |
| 模糊 | 1 | 认识但不熟 | 3天后 |
| 掌握 | 2 | 完全掌握 | 7天后 |

### 1.3 进度更新逻辑

```java
// MiniProgramController.updateProgress()
if (status == 1) nextReviewTime = now + 3天;
if (status == 2) nextReviewTime = now + 7天;

// Upsert: 存在则更新，不存在则插入
if (existing != null) updateById(progress);
else save(progress);

// 记录学习历史
studyHistoryService.recordHistory(userId, cardId, status);
```

### 1.4 复习推荐逻辑

```java
// 获取已到复习时间的卡片
List<BizUserProgress> progressList = progressService.lambdaQuery()
    .le(BizUserProgress::getNextReviewTime, LocalDateTime.now())
    .list();

// 每日推荐：每个科目随机2张
for (BizSubject subject : subjects) {
    List<BizCard> cards = cardService.lambdaQuery()
        .eq(BizCard::getSubjectId, subject.getSubjectId())
        .list();
    Collections.shuffle(cards);
    // 取前2张
}
```

### 1.6 重要说明

当前实现为**简化版固定间隔**，非真正的艾宾浩斯算法：

## 二、卡片管理模块 (Card Management)

### 2.1 核心数据表

**biz_card 表结构：**
```sql
biz_card (
  card_id BIGINT PRIMARY KEY,
  subject_id BIGINT NOT NULL,          -- FK to biz_subject
  front_content TEXT NOT NULL,          -- 题目面 (Markdown/LaTeX)
  back_content TEXT NOT NULL,           -- 答案面
  difficulty_level TINYINT DEFAULT 1,  -- 难度1-5
  audit_status CHAR(1) DEFAULT '1',    -- 0=待审核, 1=已通过, 2=已拒绝

  audit_user_id BIGINT,                -- 审核人ID
  audit_time DATETIME,                 -- 审核时间
  audit_remark VARCHAR(500),            -- 审核备注
  del_flag CHAR(1) DEFAULT '0',
  create_time DATETIME,
  update_time DATETIME
)
```

**biz_card_draft 表（用户提交草稿）：**

```sql
biz_card_draft (
```

### 2.2 审核流程

```

    ↓
保存到 biz_card_draft (status=0 待审核)
    ↓
管理员审核
```

## 四、权限系统模块 (Auth & Permissions)

### 4.1 JWT认证

**配置 (application.yml)：**
```yaml
jwt:
  secret: card-learn-jwt-secret-key-2024
  expiration: 86400000  # 24小时
  header: Authorization
  prefix: Bearer
```

**认证流程：**
```
请求 → JwtAuthenticationFilter.doFilterInternal()
    → 从Header提取 "Bearer {token}"
    → jwtUtil.validateToken(token)
    → 创建 UsernamePasswordAuthenticationToken(ROLE_USER)
    → SecurityContextHolder.setAuthentication()
    → 继续过滤器链
```

**安全配置 (SecurityConfig)：**
- CORS: 允许所有来源（开发环境）
- CSRF: 禁用（无状态API）
- Session: STATELESS无服务器会话
- 表单登录/HTTP Basic: 禁用

### 4.4 验证码系统

**生成流程：**
```java
// 1. 生成4位随机码
String code = CaptchaUtil.generateCode(); // "7K3X"

// 2. 生成UUID key
String key = UUID.randomUUID().toString();

// 3. Redis存储 (120秒TTL)
redisTemplate.opsForValue().set("captcha:" + key, code, 120, TimeUnit.SECONDS);

// 4. 生成干扰图片
BufferedImage image = CaptchaUtil.generateImage(code);

// 5. 返回 Base64 PNG
CaptchaVO { key: "uuid...", image: "data:image/png;base64,..." }
```

**验证流程：**
```java
String cachedCaptcha = redisTemplate.opsForValue().get("captcha:" + key);
if (cachedCaptcha == null) return error("验证码已过期");
if (!cachedCaptcha.equalsIgnoreCase(captcha)) return error("验证码错误");
redisTemplate.delete("captcha:" + key); // 单次使用
```

**安全特性：**
- 120秒TTL防止暴力破解
- 大小写不敏感
- 单次使用
- 干扰线和噪点防OCR

### 5.3 统计SQL查询

**总体统计：**
```sql
SELECT
    (SELECT COUNT(*) FROM biz_card WHERE del_flag = '0') AS totalCards,
    COUNT(DISTINCT DATE(p.update_time)) AS learnDays,
    COUNT(*) AS totalLearnRecords,
    SUM(CASE WHEN p.status = 0 THEN 1 ELSE 0 END) AS unlearnedCount,
    SUM(CASE WHEN p.status = 1 THEN 1 ELSE 0 END) AS fuzzyCount,
    SUM(CASE WHEN p.status = 2 THEN 1 ELSE 0 END) AS masteredCount,
    CASE
        WHEN COUNT(*) > 0
        THEN ROUND((SUM(CASE WHEN p.status >= 1 THEN 1 ELSE 0 END) * 100.0 / COUNT(*)), 1)
        ELSE 0
    END AS learnedRate
FROM biz_user_progress p
```

**科目统计：**
```sql
SELECT
    s.subject_id, s.subject_name, m.major_name,
    COUNT(DISTINCT c.card_id) AS totalCards,
    SUM(CASE WHEN p.status IS NULL OR p.status = 0 THEN 1 ELSE 0 END) AS unlearnedCount,
    SUM(CASE WHEN p.status = 1 THEN 1 ELSE 0 END) AS fuzzyCount,
    SUM(CASE WHEN p.status = 2 THEN 1 ELSE 0 END) AS masteredCount
FROM biz_subject s
LEFT JOIN biz_major m ON s.major_id = m.major_id
LEFT JOIN biz_card c ON c.subject_id = s.subject_id AND c.del_flag = '0'
LEFT JOIN biz_user_progress p ON c.card_id = p.card_id
GROUP BY s.subject_id, s.subject_name, m.major_name
HAVING totalCards > 0
```

### 5.4 用户排名算法

```sql
ORDER BY masteredCount DESC, totalCards DESC
-- 优先按掌握数降序，再按学习总数降序
```

| 功能 | 算法 |
|------|------|

### 6.1 反馈表 (biz_feedback)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键 |
| user_id | BIGINT | 用户ID |
| card_id | BIGINT | 关联卡片ID |
| major_id | BIGINT | 专业ID |

| type | VARCHAR(20) | SUGGESTION/ERROR/FUNCTION/OTHER |
| rating | TINYINT | 评分1-5星 |
| content | TEXT | 反馈内容 |
| contact | VARCHAR(100) | 联系方式 |
| images | TEXT | 图片URL (JSON数组) |

| create_time | DATETIME | 提交时间 |
| update_time | DATETIME | 处理时间 |

### 6.2 评论表 (biz_card_comment)

| 字段 | 类型 | 说明 |
|------|------|------|
| comment_id | BIGINT | 主键 |
| card_id | BIGINT | 卡片ID |
| user_id | BIGINT | 用户ID |
| user_nickname | VARCHAR(50) | 用户昵称(冗余) |
| content | TEXT | 评论内容 |
| rating | INT | 评分1-5 (默认5) |
| comment_type | VARCHAR(20) | QUALITY/POOR/NEUTRAL |
| status | VARCHAR(2) | 0=正常, 1=已处理, 2=已隐藏 |
| admin_reply | TEXT | 管理员回复 |
| feedback_id | BIGINT | 关联反馈ID |
| create_time | DATETIME | 创建时间 |

### 6.3 POOR评论自动反馈机制

当用户标记"劣质内容"时：
1. 评论保存为 `comment_type=POOR`
2. 自动创建 `biz_feedback` (type=ERROR)
3. 评论的 `feedback_id` 关联到此反馈

### 6.4 反馈状态流程

```
提交 → status=0 (待处理)
         ↓
   ┌────┴────┐
采纳(1)   忽略(2)
+回复      +回复
```

## 七、其他功能模块

### 7.1 专业与科目管理

**专业表 (biz_major)：**
- 存储考研专业（如"408计算机"）

**科目表 (biz_subject)：**
- 关联专业 (major_id)
- 存储科目（如"数据结构"、"操作系统"、"计算机网络"、"计算机组成原理"）

**层级关系：** 专业 → 科目 → 卡片

### 7.2 标签系统

**标签表 (biz_tag)：**
- 可按科目分类 (subject_id)
- 支持多对多关联卡片

**关联表 (biz_card_tag)：**
- 卡片与标签的多对多关系

### 7.3 冲刺配置

**配置表 (sys_config)：**
- 存储冲刺时间等系统配置
- 小程序端获取显示倒计时

### 7.4 日志系统

| 表 | 用途 |
|---|------|
| sys_oper_log | 操作日志 |
| sys_request_log | 请求日志 |

**请求日志记录：**
- `RequestLogAspect` 切面
- 记录所有非GET请求
- 从JWT提取用户信息

## 十一、总结

该项目是一个功能完整的考研408学习卡片系统，包含：

1. **完整的学习流程**：从浏览卡片到学习进度跟踪，再到复习提醒
2. **UGC内容生态**：用户可提交卡片，通过审核后成为正式内容
3. **多端支持**：小程序面向学习者，管理后台面向运营者
4. **数据分析**：提供学习趋势、排名、科目统计等可视化
5. **互动反馈**：评论、评分、反馈形成完整的改进闭环

技术架构清晰，采用主流的Spring Boot + Vue3 + MyBatis Plus组合，适合作为学习项目参考。
